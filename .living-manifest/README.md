# Living Manifest ✨

> 文档治理协议引擎 × 实时协作编辑器

---

## 🎯 一句话介绍

让你的 AI 助手**主动维护架构一致性**，所有变更走提案流程，对话结束文档已完整。

---

## ⚡ 快速上手

```bash
# 1. 复制到项目根目录
cp -r .living-manifest/ .claude/ 你的项目/

# 2. 启用 Git 钩子
chmod +x .living-manifest/hooks/pre-commit
cp .living-manifest/hooks/pre-commit .git/hooks/

# 3. 在项目目录启动 Claude Code
cd 你的项目 && claude
```

> 🎉 **30 秒搞定！** 协议自动加载，AI 开始静默监控。

## 🧩 核心能力

### 🛡️ 文档治理

| 冲突类型 | AI 处理方式 |
|:---------|:------------|
| 架构冲突 | 自动提案 |
| 原则变更 | 自动提案 |
| 模块边界冲突 | 自动提案 |
| 技术债务升级 | 更新攻坚清单 |
| 规则冲突 | 检测并拒绝 |
| 假设打破 | 触发规则 A |

> **关键原则**：冲突检测 100% 是 AI 的职责，**你只管说需求**。

---

### 🔄 实时协作（Live Sync）

对话中产生的想法，**自动写入文档**：

```
【同步到: docs/模块/规则推演.md】
[AI 的想法 / 分析 / 结论直接写入]
```

| 内容类型 | 同步目标 |
|:---------|:---------|
| 🆕 新想法 | 对应模块文档 |
| ✅ 结论 | 总纲或模块 |
| ⚠️ 风险 | 技术攻坚清单 |
| 🔀 架构变更 | 提案目录 |

**同步控制**：  
- `SYNC ON` / `SYNC OFF` — 开启 / 关闭实时同步  
- `NOSYNC` — 本次不同步

---

### 🔍 静默监控

AI 主动检测你的每句话是否与现有架构冲突：

- 架构冲突 → 立即提案
- 原则变更 → 提案
- 技术债务 → 归类提取

**你不需要判断，AI 100% 负责。**

## 💭 思维原则

> 每次文档操作前，必须遵循：

| 原则 | 说明 |
|:-----|:-----|
| 💡 **假设显式化** | 不隐藏困惑，明确声明假设和不确定性 |
| 📦 **最小文档** | 只解决当前问题，不做推测性扩展 |
| 🎯 **精准改动** | 只触碰必须改动的部分 |
| ✅ **目标驱动** | 先定义验收标准，再展开内容 |

---

## 🚀 协作指令

| 指令 | 作用 |
|:-----|:-----|
| `SYNC ON/OFF` | 开启 / 关闭实时同步 |
| `NOSYNC` | 本次不同步 |
| `STATUS` | 查看全局状态 |
| `PROPOSE` | 手动触发提案 |
| `APPROVE [编号]` | 通过提案 |
| `REJECT [编号]` | 拒绝提案 |
| `MODIFY [编号]` | 修改提案 |
| `FREEZE [模块名]` | 冻结模块 |
| `DIFF [版本A] [版本B]` | 查看差异 |

---

## 📋 快速开始

1. 编辑 `docs/模块/[你的模块].md` 进行设计探索
2. AI 检测到总纲变更需求 → 自动生成提案到 `docs/提案/`
3. 使用 `APPROVE [编号]` 审核通过
4. 总纲自动更新，变更历史记录到 `变更日志.md`
