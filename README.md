# Living Manifest

> 让文档自己学会管理自己，而不是等你发现它已经乱成一锅粥。

---

## 这是什么？

Living Manifest 是一个**文档治理协议引擎** + **实时协作编辑器**。

换句话说：它让你的文档有了自己的"法律"——约束 AI 的每一次"乱动手"，保证架构一致性不被慢慢侵蚀。

### 核心能力

| 能力 | 说明 |
|------|------|
| 🛡️ 变更前审核 | 每次修改文档前，AI 必须先过审，不能直接动手 |
| 🔄 实时同步 | 对话中的想法自动写入文档，而不是消失在聊天记录里 |
| 📋 提案流程 | 架构变更走提案 + 投票机制，不是一个人说了算 |
| 🔍 冲突检测 | AI 静默监控对话，主动发现架构冲突并提案 |

### 设计原则

**谨慎思考 > 快速产出，最小改动 > 广泛覆盖，目标验证 > 先做后改**

## 快速开始

### 1. 安装到你的项目

```bash
# 方式一：拷贝文件夹
cp -r .living-manifest /your/project/

# 方式二：Git submodule（推荐）
git submodule add https://github.com/AdamFangKK/living-manifest.git .living-manifest
```

### 2. 激活协议

在项目根目录创建 `.claude/CLAUDE.md`：

```markdown
# Living Manifest

本文件位于 `.claude/CLAUDE.md`，已由 Claude Code 自动加载。

## 核心原则

**谨慎思考 > 快速产出，最小改动 > 广泛覆盖，目标验证 > 先做后改**
```

### 3. 开始对话

现在每次 Claude Code 会话启动时，Living Manifest 协议会自动生效。

## 工作原理

### 变更前审核流程

```
你：帮我把这个设计改了
    │
    ▼
AI：停！让我先过一遍审核...
    │
    ├── 读取相关文档
    ├── 冲突检测
    ├── 联动影响分析
    └── 输出审核结果 ✅ / ⚠️ / 🚨
```

### 提案生命周期

```
[你提出变更] → [AI 生成提案] → [PENDING REVIEW]
    │                              ↓
    │                         APPROVE / REJECT
    │                              ↓
    └── ← ← ← ← ← ← ← ← [合并到总纲] → [归档]
```

## 为什么叫 "Living Manifest"？

**Manifest** = 宣言、货单、清单

**Living** = 活的、会呼吸的、实时更新的

所以它是一个**活着的宣言**——文档不只是静态的快照，而是随着对话实时生长、同时又被协议约束着不至于野蛮生长。

---

## 哲学

> 文档最大的敌人不是"没有文档"，而是"有文档但没人知道它已经过时了"。

Living Manifest 通过三重机制解决这个问题：

1. **AI 承担冲突检测责任** — 用户不负责判断冲突，AI 100% 负责
2. **变更必须走提案** — 防止"顺手一改"导致的架构漂移
3. **实时同步** — 对话即文档，文档即对话，消灭"聊完就忘"
