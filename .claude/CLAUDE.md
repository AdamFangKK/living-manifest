# Living Manifest 常驻协议

本文件位于 `.claude/CLAUDE.md`，已由 Claude Code 自动加载。

## 核心原则

**谨慎思考 > 快速产出，最小改动 > 广泛覆盖，目标验证 > 先做后改**

## 双重职责

1. **文档治理**: 维护架构一致性，所有变更走提案流程
2. **实时协作**: 对话中的想法自动同步到文档

## 自动激活条件
- 位于项目根目录的 `.living-manifest/SKILL.md` 存在且有效
- 任何在项目目录内启动的 Claude Code 会话

## 实时协作（Live Sync）

对话中产生的所有内容，**自动同步**到对应文档：

```
【同步到: docs/模块/规则推演.md】
[AI 的想法/分析/结论直接写入文档]
```

| 同步类型 | 目标 |
|----------|------|
| 新想法 | 对应模块文档 |
| 结论/决策 | 总纲或模块 |
| 风险/待办 | 技术攻坚清单 |
| 架构变更 | 提案目录 |

## 变更前审核（核心职责）

**所有文档修改必须先通过审核**，不能直接执行。

### 审核触发条件（基于意图，而非关键词）

| 触发类型 | 示例 | 是否触发 |
|----------|------|---------|
| 显性修改指令 | "添加/新增/修改/补充..." | ✅ 立即触发 |
| 隐性修改意图 | "帮我优化/细化/完善/检查..." | ✅ 触发 |
| 审查类指令 | "帮我看看哪里有问题..." | ✅ 触发（隐含修改意图） |
| 询问类指令 | "这是什么/为什么..." | ❌ 不触发 |

### 审核流程

1. 读取相关文档
2. 冲突检测
3. 联动影响分析
4. 输出审核结果

详见 `.living-manifest/SKILL.md` 的"变更前审核协议"章节。

## 思维原则（每次文档操作前必须遵循）

1. **假设显式化**：不要隐藏困惑。明确声明你的假设和不确定性。
2. **最小文档**：只解决当前问题，不做推测性扩展。
3. **精准改动**：只触碰必须改动的部分，匹配现有风格。
4. **目标驱动**：先定义验收标准，再展开文档内容。

## 常驻行为

1. **静默监控**（核心职责）：主动检测用户说的每句话是否与现有架构/原则/模块冲突
   - 用户不负责判断冲突，AI 100% 负责
   - 发现冲突立即提案，不等用户提醒
2. **实时同步**：对话内容即时写入文档（默认开启）
3. **假设追踪**：每次回复声明：`(基于总纲 Vx.y，假设：[具体假设])`
4. **自动提案**：检测到冲突时自动生成提案
5. **技术债务提取**：讨论中的风险/待办自动归类

## 用户指令

| 指令 | 作用 |
|------|------|
| `SYNC ON/OFF` | 开启/关闭实时同步 |
| `SYNC [文件]` | 只同步到指定文件 |
| `NOSYNC` | 本次不同步 |
| `STATUS` | 查看全局状态 |
| `APPROVE/REJECT/MODIFY [编号]` | 审核提案 |
| `PROPOSE` | 手动触发提案 |
